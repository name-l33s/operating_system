# 实验3 熟悉Linux文件系统调用

## 一、课程设计题目

实验3 熟悉Linux文件系统调用

## 二、程序功能及设计思路

### 2.1 程序功能

本实验实现了一个基于Linux系统调用的文件工具，提供以下功能：
1. 创建新文件
2. 写文件（可变记录结构）
3. 读文件
4. 修改文件权限
5. 查看文件权限
6. 退出程序

### 2.2 各功能模块设计

**1. 创建新文件**
- 使用`open()`系统调用，设置`O_CREAT`、`O_WRONLY`、`O_TRUNC`标志
- 设置默认权限为0644
- 出错处理：使用`perror()`输出错误信息

**2. 写文件**
- 使用`open()`系统调用以追加模式打开文件
- 使用`write()`写入可变记录结构
- 记录包含：ID、姓名、年龄、成绩
- 关闭文件释放资源

**3. 读文件**
- 使用`open()`系统调用以只读模式打开文件
- 使用`read()`循环读取记录
- 格式化输出记录内容
- 统计并显示记录总数

**4. 修改文件权限**
- 使用`chmod()`系统调用修改文件权限
- 用户以八进制输入权限模式（如644、755）

**5. 查看文件权限**
- 使用`fork()`创建子进程
- 子进程使用`execv()`执行`/bin/ls -l`命令
- 父进程使用`wait()`等待子进程结束

### 2.3 系统调用与C库函数的区别

| 特性 | 系统调用 | C库函数 |
|------|---------|--------|
| 实现层级 | 内核层，直接与操作系统交互 | 用户层，封装系统调用 |
| 性能 | 开销较大，模式切换 | 开销较小，可能有缓冲 |
| 可移植性 | 操作系统相关 | 标准C库，可移植性好 |
| 缓冲 | 无缓冲，直接操作 | 通常有缓冲（如stdio） |
| 示例 | open(), read(), write() | fopen(), fread(), fwrite() |

## 三、数据结构及算法设计

### 3.1 可变记录文件的记录结构

```c
typedef struct {
    int id;         // 记录ID
    char name[50];  // 姓名
    int age;        // 年龄
    float score;    // 成绩
} Record;
```

记录采用定长结构，但可以追加任意多条记录，形成可变长度的文件。

### 3.2 各系统调用的参数和用法

**1. open()**
```c
int open(const char *pathname, int flags, mode_t mode);
```
- `pathname`: 文件路径
- `flags`: 打开标志（O_RDONLY, O_WRONLY, O_RDWR, O_CREAT等）
- `mode`: 文件权限（仅O_CREAT时需要）
- 返回值: 文件描述符，失败返回-1

**2. close()**
```c
int close(int fd);
```
- `fd`: 文件描述符
- 返回值: 成功返回0，失败返回-1

**3. read()**
```c
ssize_t read(int fd, void *buf, size_t count);
```
- `fd`: 文件描述符
- `buf`: 缓冲区
- `count`: 要读取的字节数
- 返回值: 实际读取的字节数，0表示EOF，-1表示错误

**4. write()**
```c
ssize_t write(int fd, const void *buf, size_t count);
```
- `fd`: 文件描述符
- `buf`: 要写入的数据
- `count`: 要写入的字节数
- 返回值: 实际写入的字节数，-1表示错误

**5. chmod()**
```c
int chmod(const char *pathname, mode_t mode);
```
- `pathname`: 文件路径
- `mode`: 新的权限模式
- 返回值: 成功返回0，失败返回-1

**6. execv()**
```c
int execv(const char *path, char *const argv[]);
```
- `path`: 可执行文件路径
- `argv`: 参数列表（以NULL结尾）
- 返回值: 成功不返回，失败返回-1

## 五、编程中遇到的困难及解决方法

### 5.1 遇到的困难

1. **系统调用参数理解**：对open()的flags参数组合不熟悉
2. **execv()参数传递**：需要正确构造参数数组，注意以NULL结尾
3. **八进制权限输入**：需要使用%o格式符读取八进制权限模式

### 5.2 解决方法

1. 查阅系统调用手册（man 2 open），理解各个标志的含义
2. 仔细构造argv数组，确保最后一个元素为NULL
3. 使用%o格式符，并提醒用户输入八进制权限

### 5.3 实习心得

通过本次实验，深入理解了Linux系统调用的工作原理。系统调用是用户程序与操作系统内核交互的桥梁，虽然使用起来比C库函数稍显复杂，但更底层、更灵活。

实验中使用的可变记录结构展示了如何将结构化数据存储到文件中。使用系统调用直接操作文件，没有C库的缓冲层，更能理解文件I/O的本质。

execv()的使用展示了如何在程序中执行外部命令，这种方式在shell脚本和系统工具中非常常见。

建议在后续实验中可以增加文件定位（lseek）、文件锁等功能，进一步学习文件系统调用的高级特性。
