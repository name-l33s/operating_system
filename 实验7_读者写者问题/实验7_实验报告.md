# 实验7 信号量与读者-写者问题的同步实现

## 一、课程设计题目

实验7 信号量与读者-写者问题的同步实现

## 二、程序功能及设计思路

### 2.1 程序功能

本实验实现了读者-写者问题的公平调度算法，包含以下特性：
1. 允许多个读者同时读取共享资源
2. 写者必须独占访问共享资源
3. 避免写者饥饿：连续3次延迟后写者优先
4. 使用信号量实现同步
5. 统计并输出平均等待时间

### 2.2 读者-写者问题的概念和同步难点

**问题描述**：
- 共享资源（如文件、数据库）
- 读者：只读，可同时访问
- 写者：修改，必须独占

**同步难点**：
1. **读者优先**：可能导致写者饥饿
2. **写者优先**：可能导致读者饥饿
3. **公平性**：需要在两者之间平衡
4. **正确性**：保证读-写、写-写互斥

### 2.3 信号量机制的原理（P/V操作）

**信号量**：
- 一种同步原语
- sem_wait (P)：请求资源，若不可用则阻塞
- sem_post (V)：释放资源，唤醒等待进程

**实现原语**：
```c
sem_init(&sem, 0, initial_value)  // 初始化
sem_wait(&sem)                    // P操作
sem_post(&sem)                    // V操作
sem_destroy(&sem)                 // 销毁
```

### 2.4 避免写者饥饿的公平调度策略

**调度策略**：
- 初始：读者优先
- 写者等待计数：writer_deferred
- 当 writer_deferred >= 3：写者优先
- 写者执行后：writer_deferred 重置

**平衡机制**：
- 既保证读者并发，又防止写者饥饿
- 3次可根据实际情况调整

## 三、数据结构及算法设计

### 3.1 同步变量的定义和初始化（macOS兼容）

由于macOS上的sem_init已废弃，本实验使用pthread互斥锁和条件变量实现同步：

```c
pthread_mutex_t mutex;       // 互斥锁
pthread_cond_t can_read;     // 读者条件变量
pthread_cond_t can_write;    // 写者条件变量
int readers_active = 0;      // 活跃读者数
int writers_active = 0;      // 活跃写者数
int readers_waiting = 0;     // 等待的读者数
int writers_waiting = 0;     // 等待的写者数
int writer_deferred_count = 0;  // 写者延迟计数（避免饥饿）
```

**初始化**：
- pthread_mutex_init(&mutex, NULL)
- pthread_cond_init(&can_read, NULL)
- pthread_cond_init(&can_write, NULL)

### 3.2 读者优先与写者优先的判断逻辑

**读者算法**：
```c
mutex.lock
readers_waiting++
// 如果写者延迟次数>=3，或者有写者活动，就等待
while writers_active > 0 or (writer_deferred_count >= 3 and writers_waiting > 0):
    cond_wait(can_read, mutex)
readers_waiting--
readers_active++
mutex.unlock
read()
mutex.lock
readers_active--
if readers_active == 0 and writers_waiting > 0:
    cond_signal(can_write)
mutex.unlock
```

**写者算法**：
```c
mutex.lock
writers_waiting++
while readers_active > 0 or writers_active > 0:
    writer_deferred_count++
    cond_wait(can_write, mutex)
writers_waiting--
writer_deferred_count = 0
writers_active = 1
mutex.unlock
write()
mutex.lock
writers_active = 0
// 优先唤醒等待的读者，除非写者已被延迟很多次
if writers_waiting > 0 and writer_deferred_count >= 3:
    cond_signal(can_write)
elif readers_waiting > 0:
    cond_broadcast(can_read)
elif writers_waiting > 0:
    cond_signal(can_write)
mutex.unlock
```

### 3.3 线程创建和同步控制

使用 pthread 库：
- pthread_create：创建线程
- pthread_join：等待线程结束
- 模拟时间：usleep(ms * 1000)

## 五、编程中遇到的困难及解决方法、实习心得

### 5.1 遇到的困难

1. **信号量使用**：正确理解P/V操作的顺序
2. **并发同步**：保证读-读并发，读-写、写-写互斥
3. **避免饥饿**：设计合适的调度策略
4. **时间模拟**：在多线程环境中准确模拟时间

### 5.2 解决方法

1. 仔细设计信号量的获取和释放顺序
2. 用 read_count 计数器管理读者并发
3. 引入 writer_deferred 计数器，超过阈值切换优先级
4. 使用 CLOCK_MONOTONIC 获得稳定时间戳

### 5.3 实习心得

通过本次实验，深入理解了信号量和同步问题。

读者-写者问题是经典的同步问题，没有完美的解决方案，需要根据应用场景在公平性和效率之间权衡。纯读者优先或纯写者优先都会导致另一方饥饿，公平调度是更好的选择。

信号量是强大的同步工具，但使用不当容易导致死锁。良好的设计和充分的测试是避免死锁的关键。

多线程编程需要仔细考虑各种竞态条件。即使逻辑看起来正确，在实际并发执行中也可能出现意想不到的问题。

建议在后续实验中可以增加更多读者和写者，或实现其他变种（如读写锁），进一步深入理解同步机制。
