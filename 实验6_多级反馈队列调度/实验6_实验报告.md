# 实验6 多级反馈队列调度

## 一、课程设计题目

实验6 多级反馈队列调度

## 二、程序功能及设计思路

### 2.1 程序功能

本实验实现了多级反馈队列（MLFQ）调度算法，包含以下特性：
1. 3级队列，优先级Q1 > Q2 > Q3
2. 时间片逐级加倍：4、8、16
3. 新进程进入最高优先级队列Q1
4. 时间片用完未结束则降级到下一队列
5. 支持紧急进程抢占
6. 优先级提升机制：低优先级队列等待超过32时间单位则提升

### 2.2 MLFQ多级反馈队列调度原理

多级反馈队列是一种综合型调度算法，结合了FCFS、RR和优先级调度的特点：

**基本思想**：
1. 设置多个就绪队列，每个队列有不同的优先级
2. 新进程进入最高优先级队列
3. 每个队列采用时间片轮转
4. 时间片用完未完成则降级到下一队列
5. 优先级高的队列时间片较短

**设计目标**：
- 短作业优先（类似SJF）
- 交互性好（I/O密集型进程保持高优先级）
- 公平性（防止饥饿）

### 2.3 时间片逐级加倍、优先级降级、紧急抢占、优先级提升的设计思想

**1. 时间片逐级加倍**
- Q1: 4，Q2: 8，Q3: 16
- 短进程在高优先级快速完成
- 长进程在低优先级获得更长时间片

**2. 优先级降级**
- 时间片用完未完成则降级
- 防止长进程占用高优先级队列

**3. 紧急进程抢占**
- 紧急进程立即进入Q1并抢占当前进程
- 被抢占进程回到原队列队尾
- 保证重要任务及时响应

**4. 优先级提升**
- 低优先级队列等待超过32时间单位则提升
- 防止饥饿，保证公平性

## 三、数据结构及算法设计

### 3.1 三级队列的数据结构定义

```c
typedef struct {
    char name;
    int arrival_time;
    int burst_time;
    int remaining_time;
    int start_time;
    int finish_time;
    int turnaround_time;
    int current_queue;  // 0=Q1, 1=Q2, 2=Q3
    int wait_in_queue;  // 在当前队列等待的时间
    bool is_urgent;
    bool completed;
    bool in_queue;
} Process;

typedef struct {
    Process *data[MAX_PROCESSES];
    int front, rear;
    int count;
} Queue;
```

### 3.2 时间片轮转与降级逻辑

```c
// 选择要执行的进程
if (!is_empty(&q1)) {
    current = dequeue(&q1);
    quantum = QUEUE1_QUANTUM;  // 4
} else if (!is_empty(&q2)) {
    current = dequeue(&q2);
    quantum = QUEUE2_QUANTUM;  // 8
} else if (!is_empty(&q3)) {
    current = dequeue(&q3);
    quantum = QUEUE3_QUANTUM;  // 16
}

// 执行进程
run_time = min(remaining_time, quantum);
if (remaining_time == 0) {
    // 进程完成
} else {
    // 降级到下一队列
    if (current_queue == 0) enqueue(&q2, current);
    else if (current_queue == 1) enqueue(&q3, current);
    else enqueue(&q3, current);
}
```

### 3.3 紧急进程抢占机制

```c
// 检查是否有紧急进程到达
Process *urgent = NULL;
for (int i = 0; i < n; i++) {
    if (processes[i].is_urgent && processes[i].arrival_time == current_time) {
        urgent = &processes[i];
        break;
    }
}

// 抢占当前进程
if (urgent && running) {
    // 将当前运行进程放回其队列头部
    enqueue_front(&q1, running);  // 或q2, q3
    running = NULL;
}
```

### 3.4 等待超时提升机制

```c
every BOOST_TIME (32) time units:
    // 将Q2中的进程提升到Q1
    while (!is_empty(&q2)) {
        Process *p = dequeue(&q2);
        p->current_queue = 0;
        enqueue(&q1, p);
    }
    // 将Q3中的进程提升到Q2
    while (!is_empty(&q3)) {
        Process *p = dequeue(&q3);
        p->current_queue = 1;
        enqueue(&q2, p);
    }
```

1. **紧急进程优先**：D(到达6)和G(到达10)立即抢占CPU，周转时间分别为3和7

## 五、编程中遇到的困难及解决方法、实习心得

### 5.1 遇到的困难

1. **多队列管理**：需要同时维护三个队列的状态，容易出错
2. **抢占时机判断**：紧急进程到达时需要正确抢占当前进程
3. **优先级提升时机**：准确计算32时间单位的提升周期
4. **进程状态管理**：确保进程不会被重复入队或遗漏

### 5.2 解决方法

1. 使用三个Queue结构分别管理Q1、Q2、Q3，每个进程记录in_queue标志
2. 实现enqueue_front函数，将被抢占的进程放回队列头部
3. 使用last_boost_time变量记录上次提升时间，current_time - last_boost_time >= 32时触发
4. 在入队前检查in_queue标志，避免重复入队

### 5.3 实习心得

通过本次实验，深入理解了多级反馈队列调度算法的工作原理。

MLFQ是一种非常精巧的调度算法，不需要预知作业运行时间，却能达到类似SJF的效果。它通过动态调整进程优先级，平衡了响应时间和吞吐量。

紧急进程抢占机制让系统能够优先处理重要任务，优先级提升机制防止了饥饿，这两个机制让MLFQ更加完善和公平。

调度算法的设计需要在多个目标之间权衡：响应时间、吞吐量、公平性、CPU利用率等。没有一种算法在所有情况下都是最优的，需要根据应用场景选择合适的算法。

建议在后续实验中可以增加更多队列，或尝试不同的时间片设置，观察调度性能的变化。也可以实现动态调整时间片的机制，进一步优化调度效果。
