# 实验2 磁盘调度

## 一、课程设计题目

实验2 磁盘调度

## 二、程序功能及设计思路

### 2.1 程序功能

本实验实现了三种经典的磁盘调度算法：先来先服务（FCFS）、最短寻道优先（SSTF）、电梯调度（SCAN）。程序输出每种算法的磁盘访问顺序和平均移动道数。

### 2.2 三种算法原理

**1. FCFS（先来先服务）**
- 按照磁盘请求到达的先后顺序进行调度
- 完全公平，不考虑磁道位置
- 优点：实现简单，请求不会饿死
- 缺点：平均寻道时间长，磁头移动频繁

**2. SSTF（最短寻道优先）**
- 每次选择当前磁头位置到目标磁道距离最短的请求
- 可以显著减少平均寻道时间
- 优点：平均寻道时间短，吞吐量高
- 缺点：可能导致某些请求饥饿，距离相同时按FCFS处理

**3. SCAN（电梯调度）**
- 磁头沿着一个方向移动，依次服务所有请求，直到该方向无请求，然后折返
- 类似电梯的运行方式
- 优点：性能较好，避免饥饿
- 缺点：两端请求等待时间可能较长

### 2.3 当前磁头位置和移动方向的影响

- **初始位置**：从90号磁道开始
- **移动方向**：SCAN算法初始向磁道号增加方向移动
- 磁头位置和移动方向直接影响调度顺序和移动道数
- 电梯算法通过限制移动方向，保证了公平性和较好的性能

## 三、数据结构及算法设计

### 3.1 磁盘请求数据结构

```c
typedef struct {
    char name;      // 请求名称
    int track;      // 磁道号
    bool visited;   // 是否已访问
} Request;
```

### 3.2 三种调度算法实现逻辑

**1. FCFS**
```
遍历请求数组，依次服务每个请求
累加移动距离
```

**2. SSTF**
```
while (还有未服务的请求) {
    遍历所有未服务请求，找到距离当前磁头最近的
    服务该请求，更新磁头位置
}
```

**3. SCAN**
```
初始方向：向上（磁道号增加）
while (还有未服务的请求) {
    在当前方向上寻找最近的未服务请求
    如果找到，服务并更新磁头位置
    如果没找到，反转方向
}
```

### 4.2 三种算法优劣比较

| 算法 | 平均移动道数 | 公平性 | 实现难度 | 饥饿问题 |
|------|-------------|--------|---------|---------|
| FCFS | 71.43 | 最好 | 最简单 | 无 |
| SSTF | 18.57 | 较差 | 简单 | 有 |
| SCAN | 23.57 | 较好 | 中等 | 无 |

**性能排序**：SSTF > SCAN > FCFS
**公平性排序**：FCFS > SCAN > SSTF

SSTF虽然平均性能最好，但在实际应用中SCAN更为常用，因为它避免了饥饿问题，且性能接近SSTF。

## 五、编程中遇到的困难及解决方法

### 5.1 遇到的困难

1. **SCAN方向切换**：如何正确判断何时切换移动方向
2. **距离相等处理**：SSTF中多个请求距离相同时，如何按FCFS顺序处理
3. **请求状态管理**：需要确保每个请求只被服务一次

### 5.2 解决方法

1. 使用`direction_up`标志位，遍历当前方向无请求时自动切换
2. SSTF中按数组顺序遍历，距离相同时选择先到达的请求
3. 添加`visited`标志位，防止重复服务

### 5.3 实习心得

通过本次实验，深入理解了磁盘调度算法对系统I/O性能的影响。发现SSTF虽然能获得最短平均寻道时间，但在实际系统中SCAN算法更为实用，因为它兼顾了性能和公平性。

实验中使用的14个请求数据很好地展示了三种算法的差异。FCFS虽然简单，但性能确实最差；SSTF性能最佳，但可能导致某些请求长时间等待；SCAN则是一个很好的折中方案。

建议在后续实验中可以增加C-SCAN（循环扫描）算法，以及考虑请求到达时间的动态调度，使实验更接近真实磁盘调度场景。
