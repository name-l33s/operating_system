# 实验1 作业调度

## 一、课程设计题目

实验1 作业调度

## 二、程序功能及设计思路

### 2.1 程序功能

本实验实现了三种经典的作业调度算法：先来先服务（FCFS）、短作业优先（SJF）、响应比高者优先（HRRN），同时实现了两道作业并发运行的模拟。程序输出每种算法的调度顺序、每个作业的周转时间和带权周转时间、以及平均周转时间和平均带权周转时间。

### 2.2 三种算法原理

**1. FCFS（先来先服务）**
- 按照作业到达的先后顺序进行调度
- 先到达的作业优先获得CPU使用权
- 优点：简单易实现，公平
- 缺点：短作业可能等待时间过长，平均周转时间较大

**2. SJF（短作业优先）**
- 每次选择当前就绪队列中运行时间最短的作业
- 可以有效降低平均周转时间
- 优点：平均周转时间最小
- 缺点：可能导致长作业饥饿，需要预先知道作业运行时间

**3. HRRN（响应比高者优先）**
- 响应比 = (等待时间 + 运行时间) / 运行时间 = 1 + 等待时间/运行时间
- 每次选择响应比最高的作业
- 是FCFS和SJF的折中方案，兼顾了长短作业
- 优点：避免长作业饥饿，性能接近SJF

### 2.3 两道作业并发模拟思路

1. **时间推进**：使用循环模拟时间流逝，每个时间单位检查作业状态
2. **槽位管理**：维护两个运行槽位，每个槽位记录当前运行作业和剩余时间
3. **作业调入**：当槽位空闲时，从就绪队列中选择最早到达的作业调入
4. **作业调出**：作业运行完成后，释放槽位并计算周转时间

## 三、数据结构及算法设计

### 3.1 作业结构体定义

```c
typedef struct {
    char name;           // 作业名称
    int arrival_time;    // 到达时间
    int run_time;        // 运行时间
    int finish_time;     // 完成时间
    int turnaround_time; // 周转时间
    double weighted_turnaround_time; // 带权周转时间
    bool completed;      // 是否完成
} Job;
```

### 3.2 三种调度算法选择逻辑

**1. FCFS**：遍历作业数组，在所有已到达且未完成的作业中选择到达时间最早的
```c
for each uncompleted + arrived job:
    if arrival_time < earliest:
        earliest = arrival_time, selected = this job
```

**2. SJF**：在已到达且未完成的作业中，选择运行时间最短的
```c
for each uncompleted + arrived job:
    if run_time < shortest:
        shortest = run_time, selected = this job
```

**3. HRRN**：在已到达且未完成的作业中，计算响应比并选择最高的
```c
for each uncompleted + arrived job:
    ratio = 1 + (current_time - arrival_time) / run_time
    if ratio > highest_ratio:
        highest_ratio = ratio, selected = this job
```

### 3.3 双道作业并发模拟算法

```c
while (done < n) {
    // 处理作业完成（剩余时间为0的作业）
    for each slot:
        if remaining == 0: 标记完成，释放槽位

    // 填充空闲槽位
    for each free slot:
        选择最早到达的未运行作业，调入

    // 执行一个时间单位
    for each occupied slot:
        remaining -= 1

    current_time++
}
```

### 4.2 三种算法优劣比较

| 指标 | FCFS | SJF | HRRN |
|------|------|-----|------|
| 平均周转时间 | 72.90 | 56.70 | 57.30 |
| 平均带权周转时间 | 5.40 | 2.94 | 3.08 |
| 公平性 | 最好 | 较差 | 较好 |
| 实现复杂度 | 最简单 | 简单 | 较复杂 |
| 饥饿问题 | 无 | 长作业可能饥饿 | 无 |

**性能排序**：SJF > HRRN > FCFS
**公平性排序**：FCFS > HRRN > SJF

- FCFS 实现最简单，公平性最好，但平均周转时间最长，尤其短作业等待时间长
- SJF 平均性能最优，但可能导致长作业饥饿（如 E 等待 153 时间单位）
- HRRN 是良好的折中方案，兼顾了系统效率和公平性

## 五、编程中遇到的困难及解决方法

### 5.1 遇到的困难

1. **FCFS 的正确实现**：需要严格按到达时间排序，不能依赖数组初始顺序
2. **双道作业并发模拟**：如何正确管理两个运行槽位的状态转换和时间推进
3. **作业状态维护**：需要确保作业不会被重复调度
4. **C语言字符串操作**：拼接调度顺序字符串时需要注意缓冲区大小

### 5.2 解决方法

1. 在 FCFS 中遍历所有已到达作业，选择到达时间最早的，而非按数组顺序选择
2. 使用"填充后执行"的时间推进模型：先检查完成、再填充空闲槽位、最后执行一个时间片
3. 添加`completed`标志位和运行时槽位检查，防止重复调度
4. 预分配 100 字节的字符数组，使用`sprintf`+`strcat`安全拼接

### 5.3 实习心得

通过本次实验，深入理解了作业调度算法的工作原理。三种算法各自的优缺点非常明显：
- FCFS 很公平但效率低
- SJF 效率最高但可能饿死长作业
- HRRN 是实际中常用的折中方案

调度算法设计需要权衡多个目标（效率、公平、响应时间），没有一种算法在所有场景下都是最优的。双道作业的并发运行显著提升了系统吞吐量，体现了多道程序设计的优势。

建议在后续实验中可以增加更多调度算法（如优先级调度、多级反馈队列），并考虑作业 I/O 时间的模拟，使实验更接近真实场景。
