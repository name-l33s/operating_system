# 实验1 作业调度

## 一、课程设计题目

实验1 作业调度

## 二、程序功能及设计思路

### 2.1 程序功能

本实验实现了三种经典的作业调度算法：先来先服务（FCFS）、短作业优先（SJF）、响应比高者优先（HRRN），同时实现了两道作业并发运行的模拟。程序输出每种算法的调度顺序、每个作业的周转时间和带权周转时间、以及平均周转时间和平均带权周转时间。

### 2.2 三种算法原理

**1. FCFS（先来先服务）**
- 按照作业到达的先后顺序进行调度
- 先到达的作业优先获得CPU使用权
- 优点：简单易实现，公平
- 缺点：短作业可能等待时间过长，平均周转时间较大

**2. SJF（短作业优先）**
- 每次选择当前就绪队列中运行时间最短的作业
- 可以有效降低平均周转时间
- 优点：平均周转时间最小
- 缺点：可能导致长作业饥饿，需要预先知道作业运行时间

**3. HRRN（响应比高者优先）**
- 响应比 = (等待时间 + 运行时间) / 运行时间
- 每次选择响应比最高的作业
- 是FCFS和SJF的折中方案，兼顾了长短作业
- 优点：避免长作业饥饿，性能接近SJF

### 2.3 两道作业并发模拟思路

1. **时间推进**：使用循环模拟时间流逝，每个时间单位检查作业状态
2. **槽位管理**：维护两个运行槽位，每个槽位记录当前运行作业和剩余时间
3. **作业调入**：当槽位空闲时，从就绪队列中选择作业调入
4. **作业调出**：作业运行完成后，释放槽位并计算周转时间

## 三、数据结构及算法设计

### 3.1 作业结构体定义

```c
typedef struct {
    char name;           // 作业名称
    int arrival_time;    // 到达时间
    int run_time;        // 运行时间
    int start_time;      // 开始时间
    int finish_time;     // 完成时间
    int turnaround_time; // 周转时间
    double weighted_turnaround_time; // 带权周转时间
    bool completed;      // 是否完成
} Job;
```

### 3.2 三种调度算法选择逻辑

1. **FCFS**：遍历作业数组，选择第一个到达且未完成的作业
2. **SJF**：在已到达且未完成的作业中，选择运行时间最短的
3. **HRRN**：在已到达且未完成的作业中，计算响应比并选择最高的

### 3.3 时间模拟主循环设计

```
while (未完成作业数 < 总作业数 || 有作业在运行) {
    更新运行中作业的剩余时间
    检查是否有作业完成，完成则释放槽位并计算指标
    检查是否有空闲槽位，有则调入新作业
    时间 + 1
}
```

## 四、程序运行情况

### 4.1 运行结果

#### FCFS 先来先服务
- **调度顺序**: A B C D E F G H I J 
- **平均周转时间**: 86.50
- **平均带权周转时间**: 7.01

#### SJF 短作业优先
- **调度顺序**: A G F B I J C H D E 
- **平均周转时间**: 56.70
- **平均带权周转时间**: 2.94

#### HRRN 响应比高者优先
- **调度顺序**: A B G I F J C H D E 
- **平均周转时间**: 57.30
- **平均带权周转时间**: 3.08

#### FCFS 先来先服务(双道作业)
- **调度顺序**: A B C D F G E I H J 
- **平均周转时间**: 44.80
- **平均带权周转时间**: 3.47

### 4.2 三种算法优劣比较

1. **平均周转时间**：SJF < HRRN < FCFS
2. **公平性**：FCFS > HRRN > SJF
3. **实现复杂度**：FCFS最简单，SJF和HRRN需要额外计算
4. **适用性**：
   - FCFS适用于作业运行时间相差不大的情况
   - SJF适用于短作业居多的场景
   - HRRN是两者的折中，综合性能较好

双道作业并发运行有效降低了平均周转时间，提高了系统资源利用率。

## 五、编程中遇到的困难及解决方法

### 5.1 遇到的困难

1. **双道作业并发模拟**：如何正确管理两个运行槽位的状态转换
2. **作业状态维护**：需要确保作业不会被重复调度
3. **C语言字符串操作**：拼接调度顺序字符串时需要注意缓冲区大小

### 5.2 解决方法

1. 使用数组维护运行槽位状态，每个时间单位更新剩余时间
2. 添加`completed`标志位，防止重复调度
3. 预分配足够大的字符数组，使用`sprintf`安全拼接

### 5.3 实习心得

通过本次实验，深入理解了作业调度算法的工作原理。发现SJF虽然平均性能最好，但在实际应用中HRRN更为实用，因为它不需要预知作业运行时间，且避免了长作业饥饿。双道作业的并发运行显著提升了系统吞吐量，体现了多道程序设计的优势。

建议在后续实验中可以增加更多调度算法（如优先级调度），并考虑作业I/O时间的模拟，使实验更接近真实场景。
